Terms for composition, compression, production, closure, recursion, forcing.

PRODUCTIVE RETURN (U7)

Grid home: B(0, P1) Type: D — unnamed primitive Status of defining claim: FORCED

Definition: Self-action that generates genuinely new content. The structural act in which f applied to its own output produces more than was present in the input. The framework realization: R²=R+I — the return (R²) exceeds the departure (R) by the identity (I).

Tower ascent: - Level 0: Mode (iv) of Thm 0.3c — productive recurrence as one of four exhaustive self-action modes - Level 3: R²=R+I — the Fibonacci matrix equation; eigenvalue φ governs growth rate - Level 4: Möbius-RG r(n+1)=1/(1+r(n)) — each iteration returns with new content (convergents of φ̄) - Level 5: K6’ loop K→F→U(K)→K — the observer returns to itself having generated the universe-model - Level 7: SIL discovery operator M — each pass through the framework generates updated status assignments

Lift mechanism: Monoidal lift T(n)⊗T(n). The self-product of productive return is productive return at the next level.

Cross-references: Recursive completion (U2), Closure (contranym), R(R)=R Usage discipline: Use when self-action generates content. Do not conflate with cyclic return (mode (ii), J²=I) which generates nothing new.

Grid home: B(0, P1) Type: D — unnamed primitive Status: FORCED

Definition: Closure that simultaneously terminates one process and feeds the next. The two-phase operator: Phase 1 — stabilization at the current level (terminal). Phase 2 — the stabilized object becomes input at the next level (gateway).

Tower ascent: - Level 2: q∘q=q — the quotient stabilizes; the stable quotient is what K6’ then operates on - Level 5: K6’ closes the observer loop; the closed loop is what K7’ encodes - Level 7: Status(Status(S))=Status(S) — the SIL stabilizes; the stable classification is what the discovery operator acts on - Level 8: The Dictionary stabilizes term meanings; the stable meanings are what theorem statements use

Cross-references: Closure (contranym — recursive completion IS the gateway face of closure) Usage discipline: Use when closure at level n feeds structure at level n+1. Distinguish from terminal closure (stabilization with no further reentry).

FORCING

Grid home: B(2, P1) Type: A — synonym cluster Status: FORCED

Definition: Zero-branching derivation. br_s=0 at every step, conclusion unique.

Strict synonym cluster members: forced, zero-branching, the unique survivor, absolutely forced, exact (in framework-specific sense) Common role: The derivation step admits no alternative. br_s=0 at every step.

Note on “unique”: “Unique” is a near-synonym of FORCED only in its strictly unique (zero-branching) sense. The framework also uses uniqueness in three other senses — up to equivalence, by universal property, and by exhaustive elimination — which are not synonyms of FORCED. See UNIQUE entry (DICTIONARY). In the synonym cluster, “the unique survivor” is shorthand for “strictly unique by five-way elimination,” not for uniqueness in general.

Qualified near-synonyms (require explicit subtype): derived, canonical These words are not commitments on their own. Every serious use of “derived” in theorem statements or physical-commitment prose must carry its production subtype explicitly: - strictly forced (= G.1, br_s=0) — bridges chain, Dist, four statuses - observer-forced (= G.6) — gauge connection, Yang-Mills, Einstein equations - transport-derived (= G.7) — Sakharov from P1, Koide from norm ratio - reconstruction-derived (= G.8) — spacetime from Herm(M₂(ℂ)), Born rule from Gleason - derived with external processing — Λ_QCD via RG, m_p via lattice QCD - candidate-derived — pattern identified, physical closure not yet complete

Bare “derived” without subtype is ambiguous and should not appear in theorem statements.

Three-tier verb discipline: - “Forced” — reserve for strict zero-branching inevitability (G.1), or when the exact G-class is named alongside (e.g., “observer-forced”). - “Derived” — use only when either (a) the derivation is strict and contextually explicit, or (b) the generation class is stated. - “Produces” — use for broad prose when the framework contributes structure without full closure. “Produces a candidate for α_S,” “produces inputs for Λ_QCD,” “produces the Koide shape but not absolute normalization.”

Cross-references: Canonical (contranym — strong-canonical = forced), T_GOV §1 (generation taxonomy G.0–G.9), T_SIL §1 (status grammar) Usage discipline: Use “forced” when br_s=0. Do not use bare “derived” as a commitment word when the object is RESONANT, ENCODED, or framework + standard processing. Do not use “necessary” in its philosophical sense — use only in the framework sense of “no alternative consistent with prior structure.”

CANONICAL

Definition: Structurally preferred.

Contranym tension: - Face 1 (strong-canonical = forced): No alternative exists. The bridge chain is strong-canonical: each step has br_s=0. The projections π₁, π₂ are strong-canonical: universal property admits no other maps of this type. - Face 2 (weak-canonical = naturally preferred): Alternatives exist, but one is structurally favored. The Dist-ward functor is weak-canonical: the Co-Dist-ward functor also exists, but the Dist-ward direction is natural (Thm 4.5b) while the Co-Dist-ward fails naturality. - Resolution: Root instance is the construction-dissolution asymmetry (T0 §18). Construction is canonical because it preserves algebraic structure; dissolution exists but is non-canonical because it loses structure. Strong canonicity → FORCED status. Weak canonicity → structural preference without forcing.

Cross-references: Forcing (synonym cluster), Asymmetry (root contranym) Usage discipline: Always distinguish strong-canonical from weak-canonical. In theorem statements, prefer “forced” for strong and “natural/preferred” for weak.

Route Discipline for Derivation. Every claim that the framework “derives” X must specify whether X is: (a) strictly unfolded from prior framework structure, (b) transport-identified across a lawful cross-layer map, (c) reconstructed from partial or formal data, or (d) recovered through encoded standard machinery. Two-stage ecology. Every “derivation” in the repo belongs to one of two macro-families: production-route (the object is generated inside the framework) and interpretation-route (the object is recognized, reconstructed, or encoded from a framework-generated object). The Mathematical reading produces; the Observer reading structures access; the Physical reading physically realizes; the Semantic reading names. “Derive” can appear in all four, but with different route types.

For bridge-chain steps: “strictly derive.” For physical identifications: “transport-derive” or “recognize as.” For spacetime: “reconstruct.” “Emerge” remains banned.
